面向对象

  • 对象本质上是一种特殊的数据结构

  • 如果一个变量中存的是对象的地址，那该对象的是引用类型的对象

  • 类中定义的变量称为成员变量，类中定义的方法称为成员方法

  • 一个代码文件中可以定义多个类，但只能有一个类用public修饰

  • 当堆内存中存在的对象，没有本任何变量引用时，会被判定为内存中的垃圾，java存在自动垃圾回收机制

  • this关键字

    • this就是一个变量，可以用在方法中，来拿到当前对象
    • this主要用来解决变量名称冲突问题

  • 构造器

    • 对象创建时，我们可以指定对象去调用哪个构造器
    • 类在设计时，如果不写构造器，java是会为类自动生成一个无参数构造器
    • 一旦我们定义了有参数构造器，java就不会帮我们的类自动生成无参构造器

  • 封装

    • 把数据和方法设计到类中去
    • 封装设计规范 合理隐藏，合理暴露

  • 实体javaBean （实体类）

    • 这个类中的成员变量都要私有，对外提供对应的set和get方法
    • 类中必须要有一个公共的无参构造器

  • 成员变量和局部变量的区别

    区别 成员变量 局部变量
    类中位置不同 类中，方法外 常见于方法中
    初始化值不同 有默认值，不需要初始化赋值 没有默认值，使用前必须完成赋值
    内存位置不同 堆内存 栈内存
    作用域不同 整个对象 在所属的代码块中
    生命周期不同 与对象共存亡 随着方法的调用而生，随着方法的运行结束而亡

  • static

    • 成员变量按照有无static修饰，分为 类变量(有static修饰)和实例变量。类变量会被类的全部对象共享
    • 类变量， 属于类，与类一起加载一次，在内存中只有一份， 可以被类和类的所有对象共享
    • 成员方法按照有无static修饰，分为类方法(有static修饰)和实例方法
    • 使用类方法和实例方法的注意事项
      • 类方法中可以直接访问类的成员，不可以直接访问实例成员
      • 实例方法中既可以访问类成员，也可以直接访问实例成员
      • 实例方法中可以出现this关键字，类方法中不可以出现this关键字
    • 静态代码块
      • static {…}
      • 类加载时自动执行，由于类只会加载一次，所有静态代码块也只会执行一次。用于完成类的初始化，对类变量的初始化赋值
    • 实例代码块
      • {…}
      • 每次创建对象时，执行实例代码块，并在构造器之前执行
      • 和构造器一样，都是用来完成对象的初始化的。例如用来对实例变量进行初始化赋值
    • 单例设计模式

  • 继承

    • java提供了一个关键字extends, 可以让一个类和另一个类建立父子关系

    • 子类能继承父类的非私有成员(成员变量，成员方法)

    • 子类的对象是由子类和父类共同完成的

    • 权限修饰符

      修饰符 本类中 同一个包的其它类里 任意包下的子类里 任意包下的任意类里
      private √
      缺省 √ √
      protected √ √ √
      public √ √ √ √

    • 单继承，Object类

      • java是单继承的，一个类只能继承一个直接父类，java中的类不支持多继承，但是支持多层继承
      • Object类是java所有类的父类

    • 方法重写

      • 重写后，方法的访问，java会遵循就近原则
      • 使用Override注解，他可以指定java编译器，检查我们重写的格式是否正确，代码的可读性更好
      • 子类重写父类的方法，访问权限必须大于或者等于父类该方法的权限（public > protected > 缺省）
      • 重写的方法返回值类型，必须与被重写方法的方法的返回值类型一致，或者范围更小
      • 私有方法，静态方法不能被重写，如果重写会报错

    • 子类中访问其它成员（成员方法，成员变量）的特点：就近原则

    • 子类构造器的特点：子类的全部构造器，都会先调用父类的构造器，再执行自己

      • 默认情况下，子类全部构造器的第一行都是super()（写不写都有），它会调用父类的无参构造器
      • 如果父类没有无参数构造器，我们必须在子类构造器的第一行加上super(…),去调用父类指定的有参数构造器
      • 通过super(…)调用父类构造器，通过this(…)调用自己其它构造器

  • 多态

    • 多态是在继承 / 实现 情况下的一种现象，变现为：对象多态，行为多态
    • 多态前提：有继承 / 实现关系；存在父类引用子类对象，存在方法重写
    • 多态好处 1，在多态形式下，右边的对象时解耦合的，便于扩展和维护。 2，定义方法时，使用父类类型的形参，可以接收一切子类对象，扩展性更强。
    • 多态产生的问题: 多态下不能使用子类的独有功能，只要经过类型转换才可以使用

  • final

    • 修饰类：不能被继承了
    • 修饰方法：不能被重写了
    • 修饰变量：只能被赋值一次
    • static final修饰的成员变量就被称为常量

  • 抽象类

    • 抽象类和抽象方法都是用abstract修饰的；抽象方法只有方法签名，不能写方法体；
    • 抽象类中不一定有抽象方法，有抽象方法的类一定是抽象类
    • 类该有的成员抽象类都可以有
    • 抽象类不能创建对象，仅作为父类给子类继承
    • 一个类继承抽象类，必须重写抽象类的全部抽象方法，否则这个类必须也定义成抽象类

  • 接口

    • 接口的成员变量被称为常量，接口的成员方法被称为抽象方法
    • 接口不能创建对象，接口是用来被类实现的，实现接口的类称为实现类
    • 一个类可以实现多个接口，实现类必须实现全部接口的全部抽象方法，否则实现类需要定义成抽象类
    • JDK8新增的三种形式的方法
    • 接口注意事项

    • 定义类，接口，方法时，同时声明了一个或者多个类型变量， 称为泛型类，泛型接口，泛型方法，他们统称为泛型

    • 泛型的本质，把具体的数据类型作为参数传给类型变量

常用api

  • String

    • String的对象是不可变字符串对象 每次试图改变字符串对象，实际是产生了新的字符串对象，变量每次都是指向新的字符串对象，之前的字符串对象的内容确实没有改变，因此说String的对象是不可变的
    • 以String s = “…”方式写出的字符串对象，会存储到字符串常量池， 且相同的内容的字符串只会存一份；通过 String s = new String(“…”)创建的字符串对象，每new一次都会产生一个新的对象放在堆内存中。
