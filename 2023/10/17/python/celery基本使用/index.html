celery基本使用

发表于 2023-10-17

1，celery介绍

  • 使用场景
    • 异步任务
    • 定时任务
  • 组成
    • task
      • 指的是异步任务或者定时任务
    • broker
      • 可以理解为消息中间件，用于获取异步任务或者是定时任务，形成消息队列，发送给worker处理
      • broker可以是redis，rabbitmq
    • worker
      • 获取broker中的消息进行处理，将结果发送给backend
    • backend
      • worker处理完消息之后，会将结果发送出来
      • 可以配置成发送到redis，或者其它数据库中
    • beat
      • 调用定时任务
      • 异步任务的发送不需要经过beat处理，是直接发送给broker的

    • 任务重试 任务重试的参数在@app.task中定义

      • autoretry_for表示的是某种报错情况下重试，我们定义的 Exception 表示任何错误都重试
      • default_retry_delay 表示重试间隔时长, 以秒为单位。默认值是 3 * 60s，即三分钟
      • retry_kwargs 是一个 dict，其中有一个 max_retries 参数，表示的是最大重试次数
      • retry_backoff 布尔型数据，为 True 的话，自动重试的时间间隔会成倍的增长
      • retry_backoff_max 是重试的最大的间隔时间，比如重试次数设置的很大，retry_backoff 的间隔时间重复达到了这个值之后就不再增大了

  • 消息队列 task 的队列 queue 可以是多个，处理 task 的 worker 也可以是多个，worker 可以处理任意 queue 的 task，也可以处理指定 queue 的 task

  • worker

    • 指定worker的hostname

      celery -A hunter worker -l INFO -n worker1@%h

      %h 表示主机名，包含域名在内，%n 表示仅包含主机名，%d 表示仅包含域名

    • 并发处理

      如果不指定 worker 的数量，worker 的进程会默认是所在机器的 CPU 的数量。

      我们也可以通过 concurrency 参数来指定启动 worker 的进程数。

      celery -A hunter worker –concurrency=3 -l INFO

    • 消费指定队列的task

      • celery -A hunter worker –concurrency=3 -l INFO -Q test_queue1,test_queue2

      • 指定任务运行时间数据文件

        当我们启动 beat 的时候，会发现启动的文件夹下会有 celerybeat-schedule的文件，这个是 beat 保存在本地的上一次 任务运行的时间的数据，我们也可以指定该文件的输出地址

        celery -A celery_test1 beat -l INFO -s ./celerybeat-schedule

    • contrab定义

      • minute 表示分钟，接收整数或者整数列表，范围在0-59，或者字符串表示配置的时间模式

      • hour 表示小时，接收整数或者整数列表，范围在0-23，或者接收字符串表示配置的时间模式

      • day_of_week 表示周几，接收整数或者整数列表，范围在0-6，其中周日是0，周六是6，或者接收字符串表示配置的时间模式

      • day_of_month 表示一个月的第几天，接收整数或者整数列表，范围在1-31，或者接收字符串表示配置的时间模式

      • month_of_year 表示一年的第几个月，接收整数或者整数列表，范围在1-12，或者接收字符串表示配置的时间模式
