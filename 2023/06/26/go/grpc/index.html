1,grpc框架

  • grpc是一个高性能，开源，通用的RPC框架，面向移动和HTTP/2设计

  • 文档

    • grpc (yuque.com)

  • python下的protobuf使用

    • pip install grpcio -i https://pypi.douban.com/simple

    • pip install grpcio-tools -i https://pypi.douban.com/simple

    • protobuf3有自己专门的定义格式的

    • grpcio-tools 最后一个能生成方法和类的定义的版本是 1.43.0 ，从 1.44 开始就没有定义内容了，开始动态生成。

    • 根据proto文件生成对应的python文件

  • python grpc

    • 注意python生成的pb文件存在路径问题，需要手动修改

  • go grpc

    • 下载工具

      • https://github.com/protocolbuffers/protobuf/releases
      • 用来生成go的pb

    • 下载go的依赖包

      • go install google.golang.org/protobuf/cmd/protoc-gen-go@v1.27.1
      • go install google.golang.org/grpc/cmd/protoc-gen-go-grpc@v1.1.0

    • 如果在使用的时候报错 missing method ustEmbedUnimplementedGreeterServer

      • 将ustEmbedUnimplementedGreeterServer 方法注释掉

    • python和go相互调用，但是注意要保证proto文件一致，但是注意在python中不要package这一行,或者使用option go_package = “.;proto”;

2，grpc的四种数据流
  • 简单模式
  • 服务端数据流模式
    • 客户端发起一次请求，服务端返回一段连续的数据流
    • 例子：客户端向服务端发送一个股票代码，服务端将该股票的实时数据源源不断的返回给客户端
  • 客户端数据流模式
    • 与服务端流模式相反，客户端源源不断的向服务端发送数据流
    • 例子：物联网终端向服务器报送数据
  • 双向数据流模式
    • 客户端和服务端可以向对方发送数据流
    • 例子: 聊天机器人
