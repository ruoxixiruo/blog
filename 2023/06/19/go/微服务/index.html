发表于 2023-06-19
1,微服务架构

  • 伸缩立方

    • x轴：运行多个负载均衡后的实例
    • y轴：对应用进一步分解为微服务(分库)
    • z轴：大数据量时，对数据进行分区(分表)

  • 微服务的优点

    • 解决复杂问题
      • 解决单体架构耦合度高的问题
    • 团队协作更加容易
      • 每个服务可以由一个团队独立专注开发
    • 独立部署
      • 每个服务独立部署

    • 程序扩展能力强

  • 微服务缺点

    • 规模难以界定
      • 拆分的过程中，服务的规模，存在着不同的标准
    • 增加了系统的复杂度
      • 单体架构变成了分布式系统
    • 分区数据库架构难题
      • 涉及到不同服务之间的数据调用和更新
    • 项目测试难度增加
    • 多服务修改更加困难

  • 微服务的定义

    • 在分布式环境中，将单体应用拆分为一系列服务，组成一个整体

    • 每个服务都是轻量级，单独部署，运行在自己的进程中

    • 每个服务注重自己核心能力的开发，服务组件之间采用轻量级的方式进行通信，包括但是

      不限于RESTful API

    • 按照业务边界进行划分

    • 微服务是一种编程架构思想，可以用不同的语言实现

  • 网关作用

    • 提供统一服务入口，让微服务对前台透明
    • 聚合后台的服务，节省流量，提升性能
    • 提供安全，过滤，流控等API管理功能

  • 微服务之间如何通信

    • IPC （进程间通信）
      • 同步调用 REST， RPC
      • 异步消息调用

  • 服务管理（服务的发现，识别和管理）

    • 常见的服务框架有： Zookeeper
    • 基于客户端服务的注册与发现
    • 基于服务端的服务注册与发现
    • 基于客户端和基于服务端相比，基于服务端多了一个负载均衡器

  • 服务宕机等异常处理

    • 重试机制
    • 限流
    • 熔断机制
    • 负载均衡
    • 降级(本地缓存)
